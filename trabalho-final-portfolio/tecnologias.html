Tecnologias

Ferramentas usadas no dia a dia

Android Studio

Android Studio é um ambiente de desenvolvimento integrado para desenvolver para a plataforma Android. Sendo a principal ferramenta e a mais utilizada durante o desenvolvimento.

Git | GitHub | GitLab

Ferramenta de versionamento de código indispensável para codificação profissional sendo ainda mais relevante quando essa é feita por um time.

SQL Lite | Room

São bancos de dados locais que são usados para persistência através do armazenamento no dispositivo.

Firebase

Ferramenta que possibilita monitoramento de bugs, performance e navegação pelos usuários. Além disso oferece hospedagem, laboratório de testes, gerenciamento de notificações, banco de dados em tempo real e armazenamento de arquivos na nuvem.

Android Jetpack

O Jetpack é um kit de ferramentas de API independente das versões do Android, que funciona em vários dispositivos e é compatível com versões anteriores do SO. Ele inclui um pacote de mais de 70 bibliotecas androidx.

RX Java | Courotines

Ferramentas de paralelismo, permitindo a execução de códigos assíncronos não ocupando a thread principal, permitindo que a thread principal não interrompa a execução dando agilidade e fluidez a aplicação.

Jira

Ferramenta utilizada pelo métodos ágeis para criar, registrar, acompanhar, estimar horas, registrar tempo de execução das atividades. Possui diferentes quadros incluindo kanban e permite ter uma visão clara do andamento das tarefas através de gráficos e relatórios.

Dagger | Koin

Ferramentas que facilitam a utilização da injeção de dependências. Um padrão de desenvolvimento de programas de computadores utilizado para manter baixo o nível de acoplamento entre diferentes módulos de um sistema.

Postman

Postman é uma plataforma de API para desenvolvedores projetar, construir, testar e iterar as com as APIs Rest.

JUnit

Esse framework facilita a criação e manutenção do código para a automação de testes unitários com apresentação dos resultados.

Espresso

Esse framework permite a automatização de teste na interface do usuário como teste de ações do usuário, navegação e exibição de conteúdos.

Retrofit

Retrofit transforma sua API HTTP em uma interface Java.

Bitrise | GitHub Actions

Ferramentas que permitem a automação de esteiras de desenvolvimento através da integração, implantação contínua de código e até a publicação na loja de aplicativos.

Essas foram listadas apenas algumas das ferramentas utilizadas com frequencia. Lido também com outras ferramentas que também são utilizadas no desenvolvimento android, porém com uma frequência menor.

Experiência em Linguagens de programação e marcação

A primeira linguagem de programação que tive contato na vida foi a linguagem C, quando iniciei meu estudos em algoritmos e lógica de programação, porém não passou dessa visão introdutória e a execução de trabalhos na faculdade. A base do conhecimento em algoritmo e lógica de programação foram se somando a medida que conheci outras linguagens porém muito do que sabia nessa época de sintaxe se perdeu devido a falta de prática frequente. Eu programei banco com SQL na faculdade e pode se dizer que dei um esbarrão em Python. E neste momento estou usando HTML que é uma linguagem de marcação, usando o CSS para estilizar essas páginas web que você está lendo agora e o FlexBox para permitir a posicionamento e responsividade desse site que é também um conhecimento acadêmico.

Já na minha atuação profissional contando a partir do estágio em uma empresa eu pude atuar com Java, Kotlin ambas linguagens de programação e XML como linguagem de marcação. Essas sãos as linguagens mais comuns no desenvolvimento para Android nativo. pode se dizer que do meu tempo total como programador no mercado 50% eu atuei com Java e o restante com Kotlin como linguagens de programação para Android.

Java

O Java é uma linguagem de programação orientada a objetos e é uma das linguagens mais utilizadas pelas empresas na atualidade no desenvolvimento de aplicações WEB e Mobile.

Kotlin

Kotlin é uma linguagem de programação open source multiplataforma criada pela JetBrains e amplamente usada por desenvolvedores Android. É multiparadigma, sendo totalmente orientada a objetos, mas com algumas características de linguagem funcional (como as funções lambda), com tipagem estática e executada pela Java Virtual Machine (Máquina Virtual do Java).

XML

XML é uma recomendação da W3C para gerar linguagens de marcação para necessidades especiais. É um dos subtipos da SGML capaz de descrever diversos tipos de dados. Seu propósito principal é a facilidade de compartilhamento de informações por intermédio da internet.

Experiência em arquitetura de software

Durante minha atuação com codificação Android tive a oportunidade de lidar formas diferentes de estruturar códigos dos projetos e até mesmo implementações diferentes de uma mesma arquitetura. Porém a que foi mais frenquente na grande maioria dos projetos que eu atuei foram clean archtecture e MVVM.

Clean Architecture

É uma arquitetura de software proposta por Robert Cecil Martin (ou Uncle Bob, como é mais conhecido) que tem por objetivo padronizar e organizar o código desenvolvido, favorecer a sua reusabilidade, assim como independência de tecnologia. Por mais que a Clean Architecture foi criada em meados de 2012, está repleta de princípios atemporais que podem ser aplicados independente da tecnologia utilizada e linguagem de programação.

MVVM

O MVVM é um padrão de arquitetura de desenvolvimento de software baseado em Model, View e View-Model. Conheça a seguir os três principais elementos:

  • Model: a camada de modelo é a lógica de negócios que impulsiona a aplicação;

  • View: é a interface do usuário.

  • ViewModel: as camadas ViewModel coordenam as operações entre a view e as camadas model, além disso, evocará operações sobre a camada Model quando for necessário.

Tal modelo foi criado pelo arquiteto de software do WPF e Silverlight da Microsoft, John Grossman em 2005. Desde então, ele vem sendo usado principalmente no desenvolvimento mobile.
