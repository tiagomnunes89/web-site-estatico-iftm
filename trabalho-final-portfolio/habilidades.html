Tecnologias e Ferramentas

  • Android Studio

    Android Studio é um ambiente de desenvolvimento integrado para desenvolver para a plataforma Android. Sendo a principal ferramenta e a mais utilizada durante o desenvolvimento.

  • Git | GitHub | GitLab

    Ferramenta de versionamento de código indispensável para codificação profissional sendo ainda mais relevante quando essa é feita por um time.

  • SQL Lite | Room

    São bancos de dados locais que são usados para persistência através do armazenamento no dispositivo.

  • Firebase

    Ferramenta que possibilita monitoramento de bugs, performance e navegação pelos usuários. Além disso oferece hospedagem, laboratório de testes, gerenciamento de notificações, banco de dados em tempo real e armazenamento de arquivos na nuvem.

  • Android Jetpack

    O Jetpack é um kit de ferramentas de API independente das versões do Android, que funciona em vários dispositivos e é compatível com versões anteriores do SO.

  • RX Java ou Rx Kotlin | Courotines | Flow

    Rx é uma biblioteca de programação reativa que oferece uma abordagem baseada em observáveis e operadores para lidar com fluxos de dados assíncronos. Por outro lado, Kotlin Coroutines e Flow são ferramentas nativas do Kotlin para lidar com concorrência e fluxos de dados assíncronos de forma mais simples e idiomática

  • Data Binding

    É uma biblioteca de suporte para Android que permite vincular componentes de interface do usuário (UI) em seus layouts a fontes de dados em seu aplicativo usando uma sintaxe declarativa, em vez de programaticamente.

  • Android NDK

    O Android NDK é uma ferramenta que permite a criação de aplicativos Android usando linguagens de programação nativas, como C e C++. Ele é útil para desenvolvedores que desejam otimizar o desempenho de seus aplicativos ou reutilizar bibliotecas de código existentes escritas em C ou C++.

  • Retrofit

    Retrofit transforma sua API HTTP em uma interface Java.

  • Hilt | Dagger | Koin

    Ferramentas que facilitam a utilização da injeção de dependências. Um padrão de desenvolvimento de programas de computadores utilizado para manter baixo o nível de acoplamento entre diferentes módulos de um sistema.

  • Postman

    Postman é uma plataforma de API para desenvolvedores projetar, construir, testar e iterar as com as APIs Rest.

  • JUnit

    Esse framework facilita a criação e manutenção do código para a automação de testes unitários com apresentação dos resultados.

  • Espresso

    Esse framework permite a automatização de teste na interface do usuário como teste de ações do usuário, navegação e exibição de conteúdos.

  • Bitrise | GitHub Actions | GitLab CI

    Ferramentas que permitem a automação de esteiras de desenvolvimento através da integração, implantação contínua de código e até a publicação na loja de aplicativos.

  • Charles Proxy

    É uma ferramenta de depuração de rede que permite visualizar e depurar todo o tráfego HTTP e SSL/HTTPS entre o seu computador e a Internet.

  • Mockk | Mockito

    MockK é uma biblioteca de simulação (mocking) para Kotlin que permite criar e verificar objetos simulados com facilidade. Já o Mockito é um framework de testes unitários que também auxilia na simulação de classes e controle do comportamento de métodos em projetos Java. Ambos são ferramentas valiosas para escrever testes de qualidade!

  • Robolectric

    Uma biblioteca de testes para Android que permite simular o ambiente do Android sem a necessidade de um dispositivo ou emulador real.

  • Cucumber

    É uma ferramenta de automação de testes que permite a escrita de testes automatizados em linguagem natural.

  • GitHub Copilot

    GitHub Copilot é um assistente de programação em parceria com IA que ajuda desenvolvedores fornecendo sugestões de código em tempo real e automatizando tarefas repetitivas durante a codificação.

  • Jira

    Ferramenta utilizada pelo métodos ágeis para criar, registrar, acompanhar, estimar horas, registrar tempo de execução das atividades. Possui diferentes quadros incluindo kanban e permite ter uma visão clara do andamento das tarefas através de gráficos e relatórios.

  • Confluence

    Confluence é uma ferramenta de colaboração que ajuda as equipes a criar, compartilhar e colaborar em projetos de forma eficiente. É uma plataforma de gerenciamento de conteúdo que permite a criação de páginas, blogs, documentos, entre outros.

  • Slack

    Slack é uma ferramenta de comunicação que permite a troca de mensagens instantâneas entre equipes de trabalho. É uma plataforma de colaboração que reúne todas as comunicações em um só lugar, facilitando a organização e a busca por informações.

  • Google Meet

    Google Meet é uma ferramenta de videoconferência que permite a realização de reuniões online, webinars, treinamentos e eventos virtuais. É uma plataforma de comunicação que facilita a interação entre equipes de trabalho, clientes e parceiros de negócios.

  • Google Play Console

    Google Play Console é uma plataforma de desenvolvimento de aplicativos Android que permite a publicação, distribuição e análise de aplicativos na Google Play Store. É uma ferramenta essencial para os desenvolvedores que desejam disponibilizar seus aplicativos para milhões de usuários em todo o mundo.

  • Figma

    Figma é uma ferramenta de design de interface do usuário (UI) e experiência do usuário (UX) baseada na web que permite a criação de protótipos interativos, wireframes e designs de alta fidelidade. É uma plataforma colaborativa que facilita o trabalho em equipe e a comunicação entre designers, desenvolvedores e stakeholders.

  • ZSH Terminal

    O ZSH é um shell de linha de comando avançado que oferece recursos avançados de personalização, autocompletar, atalhos de teclado e temas visuais. É uma ferramenta essencial para os desenvolvedores que desejam aumentar sua produtividade e eficiência no desenvolvimento de software.

  • GitBash

    O Git Bash é um emulador de terminal que permite a execução de comandos Git e Bash no Windows. É uma ferramenta essencial para os desenvolvedores que desejam trabalhar com repositórios Git e automatizar tarefas de desenvolvimento de software.

  • Gradle

    Gradle é uma ferramenta de automação de compilação e construção de projetos que permite a definição de tarefas, dependências e configurações de forma declarativa. É uma ferramenta essencial para os desenvolvedores que desejam automatizar o processo de compilação e distribuição de seus aplicativos.

  • Fortify

    Fortify é uma ferramenta de análise de segurança de aplicativos que identifica e corrige vulnerabilidades de segurança em códigos-fonte e binários. É uma ferramenta essencial para os desenvolvedores que desejam garantir a segurança de seus aplicativos e proteger os dados dos usuários contra ameaças cibernéticas.

  • Proguard

    ProGuard é uma ferramenta de otimização e ofuscação de código que reduz o tamanho do arquivo APK e protege o código contra engenharia reversa. É uma ferramenta essencial para os desenvolvedores que desejam proteger a propriedade intelectual de seus aplicativos e garantir a segurança dos dados dos usuários.

  • Jetpack Compose

    Não tive a oportunidade de usar profissionalmente, mas já testei criando uma aplicação simples. Jetpack Compose é uma biblioteca de UI declarativa para o desenvolvimento de aplicativos Android que permite a criação de interfaces de usuário modernas e dinâmicas com menos código.

Linguagens de programação

A primeira linguagem de programação que tive contato na vida foi a linguagem C, quando iniciei meu estudos em algoritmos e lógica de programação, porém não passou dessa visão introdutória e a execução de trabalhos na faculdade. A base do conhecimento em algoritmo e lógica de programação foram se somando a medida que conheci outras linguagens porém muito do que sabia nessa época de sintaxe se perdeu devido a falta de prática frequente. Eu programei banco com SQL na faculdade e pode se dizer que dei um esbarrão em Python. E neste momento estou usando HTML que é uma linguagem de marcação, usando o CSS para estilizar essas páginas web que você está lendo agora e o FlexBox para permitir a posicionamento e responsividade desse site que é também um conhecimento acadêmico.

Já na minha atuação profissional contando a partir do estágio em uma empresa eu pude atuar com Java, Kotlin ambas linguagens de programação e XML como linguagem de marcação. Essas sãos as linguagens mais comuns no desenvolvimento para Android nativo. pode se dizer que do meu tempo total como programador no mercado 50% eu atuei com Java e o restante com Kotlin como linguagens de programação para Android.

Java

O Java é uma linguagem de programação orientada a objetos e é uma das linguagens mais utilizadas pelas empresas na atualidade no desenvolvimento de aplicações WEB e Mobile.

Kotlin

Kotlin é uma linguagem de programação open source multiplataforma criada pela JetBrains e amplamente usada por desenvolvedores Android. É multiparadigma, sendo totalmente orientada a objetos, mas com algumas características de linguagem funcional (como as funções lambda), com tipagem estática e executada pela Java Virtual Machine (Máquina Virtual do Java).

XML

XML é uma recomendação da W3C para gerar linguagens de marcação para necessidades especiais. É um dos subtipos da SGML capaz de descrever diversos tipos de dados. Seu propósito principal é a facilidade de compartilhamento de informações por intermédio da internet.

Outras linguagens

Já tive um breve contato com C, C#, HTML, CSS, JavaScript, Dart, Python e Ruby.

Experiência em arquitetura de software

Durante minha atuação com codificação Android tive a oportunidade de lidar formas diferentes de estruturar códigos dos projetos e até mesmo implementações diferentes de uma mesma arquitetura. Porém a que foi mais frenquente na grande maioria dos projetos que eu atuei foram clean archtecture e MVVM.

Clean Architecture

É uma arquitetura de software proposta por Robert Cecil Martin (ou Uncle Bob, como é mais conhecido) que tem por objetivo padronizar e organizar o código desenvolvido, favorecer a sua reusabilidade, assim como independência de tecnologia. Por mais que a Clean Architecture foi criada em meados de 2012, está repleta de princípios atemporais que podem ser aplicados independente da tecnologia utilizada e linguagem de programação.

MVVM

O MVVM é um padrão de arquitetura de desenvolvimento de software baseado em Model, View e View-Model. Conheça a seguir os três principais elementos:

  • Model: a camada de modelo é a lógica de negócios que impulsiona a aplicação;

  • View: é a interface do usuário.

  • ViewModel: as camadas ViewModel coordenam as operações entre a view e as camadas model, além disso, evocará operações sobre a camada Model quando for necessário.

Tal modelo foi criado pelo arquiteto de software do WPF e Silverlight da Microsoft, John Grossman em 2005. Desde então, ele vem sendo usado principalmente no desenvolvimento mobile.

MVI

A arquitetura MVI, sigla para Model-View-Intent, é um padrão arquitetural específico para o desenvolvimento de aplicações Android. Ela se destaca por seguir uma abordagem de fluxo de dados unidirecional e enfatizar a separação de responsabilidades e a imutabilidade para criar um código escalável, mantível e testável. No MVI, o Model mantém o estado do aplicativo e processa os eventos, a View exibe o estado e emite eventos (Intents) para o Model, e o Intent define a ação que o usuário realiza na View. Essa estrutura promove um gerenciamento de estado mais organizado e facilita a depuração.
