Android Studio is an integrated development environment for developing for the Android platform. It is the main tool and the most used during development.

Code versioning tool essential for professional coding, even more relevant when it is done by a team.

Tool that enables monitoring of bugs, performance and navigation by users. In addition, it offers hosting, testing laboratory, notification management, real-time database and file storage in the cloud.

Experience in software architecture

During my performance with Android coding I had the opportunity to deal with different ways of structuring project codes and even different implementations of the same architecture. But the one that was most frequent in the vast majority of projects that I worked on were clean architecture and MVVM.

Clean Architecture

It is a software architecture proposed by Robert Cecil Martin (or Uncle Bob, as he is better known) that aims to standardize and organize the code developed, favor its reusability, as well as technology independence. Although Clean Architecture was created in mid-2012, it is full of timeless principles that can be applied regardless of the technology used and programming language.

MVVM

MVVM is a software development architecture pattern based on Model, View and View-Model. Meet the three main elements below:

  • Model: the model layer is the business logic that drives the application;

  • View: it is the user interface.

  • ViewModel: the ViewModel layers coordinate operations between the view and the model layers, in addition, it will evoke operations on the Model layer when necessary.

Such a model was created by the software architect of WPF and Silverlight from Microsoft, John Grossman in 2005. Since then, it has been used mainly in mobile development.

MVI

The MVI architecture, acronym for Model-View-Intent, is a specific architectural pattern for the development of Android applications. It stands out for following a unidirectional data flow approach and emphasizing the separation of responsibilities and immutability to create scalable, maintainable and testable code. In MVI, the Model maintains the state of the application and processes the events, the View displays the state and emits events (Intents) to the Model, and the Intent defines the action that the user performs on the View. This structure promotes more organized state management and facilitates debugging.
